# 🛡️ Ronin Core Engine

**Ronin Core Engine** یک پلتفرم قدرتمند، متمرکز و چندزنجیره‌ای (Multi-Chain) برای اسکن، ردیابی و مدیریت تراکنش‌های بلاکچینی است. این سیستم با تحلیل خودکار داده‌های بلاک‌چین، تطبیق با لیست‌های سیاه (AML) و مدیریت دارایی‌ها، امنیت و شفافیت عملیاتی را تضمین می‌کند.

---

## 📋 قابلیت‌های کلیدی

* **پشتیبانی از چندین شبکه:** ردیابی در شبکه‌های Ethereum, BSC, Polygon, Bitcoin, TRON, TON.
* **سیستم هوشمند AML:** بررسی خودکار آدرس‌ها جهت شناسایی نهادهای پرخطر.
* **سیستم وایت‌لیست:** مدیریت دارایی‌های مورد اعتماد برای جلوگیری از حملات Spoofing.
* **یادداشت‌گذاری تیمی:** ثبت یادداشت روی تراکنش‌ها با قابلیت تعیین سطح دسترسی (Visibility).
* **API هوشمند:** شناسایی خودکار نوع ورودی (آدرس یا هش تراکنش) از طریق وب‌هوک.

---

## 🛠 نیازمندی‌ها

برای اجرای پروژه به پیش‌نیازهای زیر نیاز دارید:

* **Node.js**: نسخه 18 یا بالاتر.
* **npm**: (همراه با Node.js نصب می‌شود).
* **SQLite3**: به صورت خودکار توسط پروژه مدیریت می‌شود.

---

## 📦 کتابخانه‌های استفاده شده

این پروژه از پکیج‌های زیر بهره می‌برد:

| کتابخانه | کاربرد |
| --- | --- |
| `express` | فریم‌ورک اصلی وب‌سرور |
| `sqlite3` | دیتابیس محلی برای ذخیره تراکنش‌ها و تنظیمات |
| `bcryptjs` | هش کردن ایمن رمز عبور کاربران |
| `jsonwebtoken` | احراز هویت و مدیریت نشست‌های کاربری |
| `crypto` | تولید شناسه‌های یکتا و ابزارهای رمزنگاری |
| `https` | برقراری ارتباط ایمن با RPCهای بلاکچینی |

## 🚀 نحوه راه‌اندازی

### ۱. نصب وابستگی‌ها

در ترمینال، داخل پوشه پروژه دستور زیر را وارد کنید:

```bash
npm install express sqlite3 bcryptjs jsonwebtoken

```

### ۲. اجرای سرور

برای شروع به کار سرور، دستور زیر را اجرا کنید:

```bash
node server.js

```

سرور به صورت پیش‌فرض روی پورت **8080** اجرا می‌شود: `http://localhost:8080`

## 🔑 تنظیمات امنیتی

* **MASTER_SECRET_KEY**: برای ثبت‌نام اولین اپراتور در مسیر `/api/auth/register` استفاده می‌شود (مقدار آن در `server.js` تعریف شده است).
* **JWT_SECRET**: کلید رمزنگاری توکن‌های نشست کاربر. *(توصیه: در محیط عملیاتی حتماً این مقدار را تغییر دهید).*
* **API_KEY**: سیستم به ازای هر کاربر یک `api_key` اختصاصی (با پیشوند `ronin_`) تولید می‌کند که برای احراز هویت در وب‌هوک الزامی است.

## ⚙️ نحوه استفاده از API

### ۱. ثبت‌نام (Register)

اپراتورها باید ابتدا با استفاده از `MASTER_SECRET_KEY` ثبت‌نام کنند:

* **POST** `/api/auth/register`
* **Body**: `{"username": "...", "password": "...", "secret_key": "..."}`

### ۲. وب‌هوک اسکن تراکنش (Webhook)

برای اسکن یک تراکنش، درخواست خود را به وب‌هوک ارسال کنید:

* **POST** `/api/v1/webhook`
* **Header**: `key: <YOUR_API_KEY>`
* **Body**: `{"data": "TX_HASH_OR_ADDRESS"}`

### ۳. دریافت رسید (Receipt)

پس از پردازش، نتیجه در مسیر زیر قابل مشاهده است:

* **GET** `/api/receipt/:id`

## ⚠️ نکات امنیتی مهم

* **حفظ حریم خصوصی:** فایل دیتابیس (`ronin_platform.db`) حاوی داده‌های حساس است. آن را هرگز در مخازن عمومی (مثل گیت‌هاب) آپلود نکنید (اطمینان حاصل کنید در `.gitignore` قرار دارد).
* **امنیت سرور:** از آنجایی که سرور روی `0.0.0.0` گوش می‌دهد، حتماً از یک فایروال برای محدود کردن دسترسی‌های غیرمجاز استفاده کنید.
* **مدیریت فرآیند:** برای محیط‌های عملیاتی (Production)، از ابزاری مثل **PM2** برای مدیریت و اجرای دائمی سرور استفاده نمایید.
